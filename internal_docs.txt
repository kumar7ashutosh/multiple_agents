Longformer: Employs a combination of sliding window (local) and global attention to handle long documents (up to 16K+ tokens) with linear memory complexity. It is ideal for document-level understanding.

Optimization Techniques
Quantization: Reducing weight precision from FP32 to INT8 or FP16 drastically lowers memory usage and speeds up inference on modern GPUs, reducing energy consumption by up to 29%.
Knowledge Distillation: Training a smaller "student" model (like TinyBERT) to mimic a larger "teacher" model (BERT) reduces parameter count while preserving performance.
Pruning: Removing redundant weights or attention heads reduces the model size and FLOPs needed for inference.
PagedAttention & KV Cache: Used in Llama 2 deployment to manage key-value cache memory efficiently, increasing throughput by over 2x compared to traditional batching.

Deployment Architecture
Inference Backends: ONNX Runtime is considered one of the best for deploying transformers at scale. NVIDIA TensorRT is used for GPU-specific acceleration.
Containerization & Orchestration: Deploying models via Docker/Kubernetes, particularly using Amazon SageMaker endpoints with specialized Large Model Inference (LMI) containers for Llama 2.
Edge Deployment: Using Quantization-Aware Training (QAT) to deploy models on resource-constrained devices, often requiring models like TinyBERT or DistilBERT.
